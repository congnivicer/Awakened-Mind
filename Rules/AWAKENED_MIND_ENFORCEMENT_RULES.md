# 🧠 Awakened-Mind-Complete Workspace — Strict Enforcement Rules

## 🚨 PRIME DIRECTIVE
**This workspace is the final, production-ready system. Any action that risks code contamination, instability, or loss of integrity is strictly forbidden.**

## 🏗️ INTEGRATION POLICY
- Only clean, debugged, and fully tested code from Roody, Cosmos, NHB Core Extension, and LLM Proxy may be integrated.
- All integration must be planned, reviewed, and approved before merging.
- No experimental, test, or legacy files from other workspaces may be copied here without explicit review and cleaning.

## 🛡️ CONTAMINATION PREVENTION
- Do not copy files blindly from other workspaces.
- Remove all test, debug, and non-production files before integration.
- No code from unrelated projects or directories may be added.
- If contamination is detected, revert immediately and document the incident.

## 🧹 CLEAN CODE REQUIREMENTS
- All code must be error-free, pass all tests, and be linted.
- No commented-out code, TODOs, or temporary fixes.
- Documentation must be up-to-date and clear.
- All dependencies must be reviewed and necessary for production.

## 🔒 PROTECTED DIRECTORIES
- Never modify `/Volumes/NHB_Workspace/Awakened-Mind-Complete` directly except through approved integration.
- Do not touch `/Volumes/Storage_for_now/` or any directory marked as protected.

## 📝 WORKFLOW ENFORCEMENT
1. **Feature Branches Only:**  
   - Create a new branch for every change.
   - Never work on `main` or `master`.

2. **Backup Before Major Changes:**  
   - Always create a backup before integration or refactoring.

3. **Testing & Review:**  
   - Run all unit and integration tests.
   - Peer review required before merging.

4. **Documentation:**  
   - Update documentation and changelogs for every change.
   - Clearly document integration steps and decisions.

5. **Integration Checklist:**  
   - All tests pass
   - No linting/type errors
   - No debug/test files
   - Documentation updated
   - Backup created
   - Changes logged

## ❌ FORBIDDEN ACTIONS
- No direct commits to production without review.
- No force pushes or history rewrites.
- No skipping tests or documentation.
- No merging unclean or untested code.
- No deletion without backup.

## 🏆 SUCCESS METRICS
- System runs error-free in production.
- All components (Roody, Cosmos, NHB Core Extension, LLM Proxy) are integrated and functional.
- No contamination or legacy/test code present.
- Documentation and changelogs are complete and accurate.

## 📝 HANDOFF PROTOCOL
- When stopping work, leave clear notes:
  - What was completed
  - What’s in progress
  - Any blockers
  - Next steps
  - Branch name and commit hash
